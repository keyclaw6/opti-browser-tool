# Research Workspace

This directory will hold sourced research outputs produced before implementation choices are accepted.

- `harnesses/`: browser-agent repository reviews, architecture comparisons, and hypotheses.
- `benchmarks/`: benchmark audits, task reviews, failure-mode coverage, and suite proposals.
- `judges/`: verifier designs, judge studies, calibration cases, and error analyses.

Research documents should distinguish observed facts, inferences, hypotheses, and accepted decisions. A promising implementation pattern is not automatically a project decision.
